**Author:** CEO
**Date drafted:** 2026-05-27
**Status:** frozen until decision day. No v0.3.0 work begins before then.

## Decision deadline

**2026-06-26** (30 days from Gumroad listing live; slip 1:1 if launch slips). On that date the user picks Branch A, B, or C from Gumroad's purchase count. No earlier optimization, no mid-window pivots — the experiment was designed to be falsifiable; respect it.

## Branch A — experiment succeeded (>=3 sales in 30 days)

**Signal:** at least 3 strangers found $49 of value in skipping the build. WTP exists. Strategy: deepen the paid product so the next $1K is achievable.

**Headline bet:** convert one-shot $49 buyers into a repeatable funnel by killing the two reasons a buyer wouldn't recommend us — stale `.dmg` and the first 10 minutes after install.

### Three prioritized features

1. **Release automation for the signed `.dmg`** (risk #4, paid-tier spec).
   - Pain: every release means manual notarize + drag-upload. At 3+ buyers/cycle the email-on-update promise becomes a chore I will skip → stale builds → refunds.
   - LTV: protects buyers from churn-via-decay; required before 2-week cadence is credible.
   - Cost: **S** (1 week). GitHub Action + notarytool + Gumroad API upload.
   - NOT building: in-app auto-update (Sparkle). L-sized, unjustified at this volume.

2. **One-screen setup doctor** (funnel drop-offs #1 + #3).
   - Pain: 40%+ bounce on permission dance and `ANTHROPIC_API_KEY not set`. Paying customers are louder when this hurts.
   - LTV: turns refundable "didn't work" into 60-second success. Defends refund rate <40%.
   - Cost: **S** (3-5 days). Tauri view checking Accessibility, Screen Recording, LLM key; deep-links System Settings for each missing piece.
   - NOT building: full settings UI, account system, telemetry.

3. **One killer recipe shipped as runnable example** (funnel §5).
   - Pain: the product is a primitive, not a destination. Buyers have nothing to show their coworker on day 2.
   - LTV: a copy-pasteable workflow ("auto-triage inbox", "fill expense reports from receipts") is the most viral artifact we can ship. Expands TAM to non-agent-builders.
   - Cost: **M** (1.5-2 weeks). One recipe end-to-end + 30s README screen recording.
   - NOT building: recipe gallery, marketplace, user-submitted recipes.

### Pricing evolution

**Stay at $49.** Three sales is signal, not data; $79 with n=3 is curve-fitting noise. Only change I'd entertain: a **$199 5-seat team SKU** *iff* one of the three buyers asked about team use unprompted. Otherwise one SKU, same price, focus conversion not ARPU.

### Distribution doubling-down

Re-run attribution at day 30. Whichever of {Show HN, agent-builder Discords, Twitter, inbound} produced >=2 of 3 sales gets a second concentrated push. Others get zero new effort. The data tells us.

## Branch B — experiment failed (0 sales in 30 days)

**Signal:** $49 WTP for a convenience-only paid tier does not exist *at our current reach*.

### The hard truth surfaced

Zero is overdetermined — price, positioning, distribution, or product. Honest read: **distribution is most likely** (funnel projected 5 buyers per 100 free users; we may not have reached 100). Doesn't matter for the next call: if we can't get 100 free users to a Gumroad page in 30 days with the channels we have, the *business* doesn't work even if the *product* does. Pivot.

### Three alternative monetization bets, ranked by ROI

1. **Consulting / integration contracts (Hypothesis D).** One contract = $2-10K. Cost: zero engineering, ~10 hrs/week outbound DMs to agent infra teams. Time to revenue: **2-8 weeks**. Scariest assumption: the author can do outbound without flinching. If no, this branch dies.

2. **SDK-for-other-agent-builders.** Reposition as a dependency (PyPI / crates.io) other agent products embed. Revenue via sponsored features / consulting. Cost: **M** (2-3 weeks API hardening). Time to revenue: months, indirect via adopter leads. Scariest assumption: any other agent builder *wants* a macOS-only dependency in their stack.

3. **Open-core paid plugins (Run Journal, per-app learning).** Cost: **L** (4-6 weeks first plugin). Time to revenue: ~5 months. Scariest assumption: the plugins solve a pain anyone has articulated. Nobody has asked for Run Journal — building it pre-emptively repeats the mistake Branch B is escaping.

**Rank:** consulting > SDK > open-core. Consulting converts the existing artifact to cash with zero new code. Open-core is last: most code, weakest demand signal.

### Give-up-and-kill criterion

**If by 2026-09-26 (4 months after day-30) cursor-pointer has not produced $249 cumulative revenue across any channel, stop.** Archive the repo, document learnings, freeze v0.3.0. $249 is the Claude Max subscription — if four months can't earn back the tool used to build it, the unit economics never close.
